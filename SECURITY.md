# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in `jinja2-fragments`, please report it privately by email:

- **Email:** sergi@cub3.net

Please include:

- A clear description of the issue
- Steps to reproduce (or a proof of concept)
- The affected version(s)
- Any suggested mitigation or fix (if available)

Please **do not** report security vulnerabilities through public GitHub issues.

## Response Process

- I will acknowledge receipt as soon as possible (typically within 72 hours).
- I will investigate and validate the report.
- If confirmed, I will work on a fix and coordinate responsible disclosure.
- After a fix is available, I may publish an advisory and release notes.

## Supported Versions

Security fixes are best-effort and are typically made on the latest released version.
